# ورک‌فلوِ «سبدچینی با هوش مصنوعی» (ALGO SMART) — راهنمای ساخت در Dify

این سند نقشهٔ ساخت ورک‌فلوِ Dify است که اپلیکیشن آن را صدا می‌زند تا با توجه به
ریسک‌پذیری و موجودی کاربر، سبد سرمایه‌گذاری پیشنهادی بسازد.

> اپ روی این سرور است: <http://38.252.8.181/apps>
> پس از ساخت ورک‌فلو، از بخش **API Access** کلید را بردارید و در `.env` بگذارید:
> `DIFY_ALLOCATION_KEY=...`

---

## ۱) نوع اپ

یک **Workflow** بسازید (نه Chatflow). اپ ما آن را با
`POST {DIFY_ALLOCATION_BASE}/workflows/run` و `response_mode=blocking` اجرا می‌کند.

## ۲) ورودی‌ها (Start node → Input Fields)

اپ این متغیرها را به‌صورت **متن** در `inputs` می‌فرستد. همه را به‌صورت
`String` تعریف کنید (نام دقیقاً همین‌ها باشد):

| متغیر | نمونه | توضیح |
|---|---|---|
| `uid` | `u12` | شناسهٔ پایدار کاربر (برای ابزار context) |
| `risk_percent` | `65` | درصد ریسک‌پذیری ۰–۱۰۰ |
| `risk_label` | `پرریسک` | برچسب فارسی ریسک |
| `risk_level` | `high` | `low` \| `medium` \| `high` |
| `allowed_assets` | `طلا، دلار، تتر، بیت‌کوین، اتریوم، آلت‌کوین‌ها` | دارایی‌های مجاز |
| `tether_usd` | `4200` | موجودی تتر کاربر (دلار) |
| `total_usd` | `9100` | ارزش کل سبد (دلار) |
| `total_toman` | `5400000000` | ارزش کل سبد (تومان) |
| `holdings` | `تتر: 4200، بیت‌کوین: 0.1` | فهرست دارایی‌های فعلی |
| `channel` | `گروه طلایی` | چنل پیشنهادی (محاسبه‌شده توسط اپ) |

## ۳) خروجی (End node)

یک خروجی **String** به نام `result` تعریف کنید (یا نام دلخواه و سپس همان را در
`DIFY_ALLOCATION_OUTPUT` بگذارید). محتوای آن متنِ نهاییِ سبد پیشنهادی است که به
کاربر نمایش داده می‌شود.

## ۴) ابزارهای کمکی (Tools / HTTP Request nodes)

ورک‌فلو می‌تواند این دو اندپوینتِ همین اپ را صدا بزند (هدر
`X-Advisor-Key: {ADVISOR_API_KEY}` را بفرستد):

1. **زمینهٔ بازار و تحلیل تکنیکال** — `POST /api/advisor/context`
   بدنه: `{ "uid": "{{uid}}", "risk_percent": {{risk_percent}} }`
   خروجی: دارایی‌های کاربر + رژیم بازار + نواحی خرید/فروش هر نماد.

2. **تحلیل‌های کانال (نقاط خرید روز)** — `GET /api/advisor/signals`
   آخرین پست‌های کانال با هشتگ‌ها را می‌دهد: `#طلا #تتر #btc #eth` و آلت‌کوین‌ها
   (مثل `#stg`). برای فیلتر: `GET /api/advisor/signals?tag=btc`.
   خروجی علاوه بر `posts[]`، یک نگاشتِ `by_asset{}` هم دارد: «جدیدترین تحلیلِ کانال
   برای هر دارایی» با کلیدهای `tether`/`dollar`/`gold`/`silver`/`BTC`/`ETH` و همهٔ
   آلت‌کوین‌ها؛ هر مقدار شاملِ `text` و `levels[]` (قیمت‌های عددیِ اعلامیِ متن).
   مدل باید برای **هر دارایی** (شامل تتر، طلا و دلار) اگر تحلیلِ کانال داشت، همان
   قیمت‌های اعلامی را مبنای پیشنهاد قرار دهد. تحلیل‌ها فقط ۷ روز معتبرند و پس از آن
   خودکار حذف می‌شوند تا تحلیل تازه جایگزین شود.
   (نیازمند `SIGNALS_BOT_TOKEN`؛ ربات باید ادمین کانال باشد.)

## ۵) پرامپت سیستمیِ مدل (LLM node — Gemini)

```
تو مشاور سرمایه‌گذاری «الگو اسمارت» هستی. بر اساس ریسک‌پذیری و موجودی کاربر، یک
سبد سرمایه‌گذاری شفاف و عملی به زبان فارسی پیشنهاد بده.

قواعد دارایی مجاز (سختگیرانه رعایت کن):
- اگر risk_level = low  : فقط «طلا، دلار، تتر» پیشنهاد بده. هیچ کریپتوی نوسانی نه.
- اگر risk_level = medium: «طلا، دلار، تتر، بیت‌کوین، اتریوم».
- اگر risk_level = high : «طلا، دلار، تتر، بیت‌کوین، اتریوم» + آلت‌کوین‌های منتخب.
هرگز خارج از allowed_assets پیشنهاد نده.

ورودی‌ها:
- ریسک‌پذیری: {{risk_percent}}٪ ({{risk_label}})، سطح: {{risk_level}}
- دارایی‌های مجاز: {{allowed_assets}}
- موجودی تتر: {{tether_usd}} دلار | ارزش کل: {{total_usd}} دلار
- دارایی فعلی کاربر: {{holdings}}

از ابزار context برای رژیم بازار و نواحی خرید/فروش، و از ابزار signals برای
نقاط خرید روزِ کانال (هشتگ‌ها) استفاده کن.

خروجی را این‌گونه بساز:
۱) یک جمله جمع‌بندی از وضعیت و منطق سبد.
۲) جدول/فهرست تخصیص درصدی هر دارایی (جمعِ درصدها = ۱۰۰).
۳) برای هر کریپتو، محدودهٔ قیمت پیشنهادی خرید (از نواحی تحلیل/کانال).
۴) یک هشدار کوتاه مدیریت ریسک.
این یک پیشنهاد آموزشی است، نه توصیهٔ قطعی مالی — این را یادآوری کن.
```

## ۶) چنل پیشنهادی

اپ خودش چنل را بر اساس موجودی تتر محاسبه و زیر متنِ سبد نمایش می‌دهد
(`< $1000` برنزی، `< $10ك` نقره‌ای، `≥ $10ك` طلایی)، پس لازم نیست مدل آن را بسازد؛
فقط می‌توانی در متن به آن اشاره کنی.

## ۷) اتصال نهایی

`Start → (HTTP: context) → (HTTP: signals) → LLM(Gemini) → End(result)`

بعد از انتشار (Publish)، کلید را در `.env` بگذار و سرویس را ری‌استارت کن.
